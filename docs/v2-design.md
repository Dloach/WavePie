# WavePie V2 设计概要

> 基于 V1 经验，完全重写 PC 端。
> 摒弃鼠标/键盘/手柄所有后备输入，BLE 是唯一输入源。

---

## 交互模型：光标准星

### 核心概念
把硬件（ESP32+MPU6050 一体）想象成一把**手枪**，屏幕上的圆形菜单想像成一个靶盘。

```
      🎯 靶盘（圆形菜单）
    ┌─────────────────┐
    │   ╭─────────╮   │
    │   │  ● ←──  │   │    ● = 光标准星
    │   │  准星    │   │       自由在内径外径之间移动
    │   ╰─────────╯   │
    └─────────────────┘
    
         ↑ 倾斜枪口
         │   12点方向
    ← 9点 ────→ 3点
         │   6点方向
         ↓
```

### 流程
1. **按下扳机**（GPIO 4）：校准零点，显示光标准星
2. **摆动枪口**（IMU roll/pitch）：准星在圆环内自由移动
3. **准星重叠扇区** → 该命令高亮
4. **松开扳机**：执行当前高亮命令

### 与 V1 的区别

| | V1 | V2 |
|--|----|----|
| 控制方式 | 扇区锁定（角度→索引） | 自由移动（准星在环内随意滑动） |
| 运动模型 | 绝对角度映射固定扇区 | 增量偏移速率控制准星位置 |
| 用户直觉 | "转到哪个扇区" | "瞄准哪个扇区" |
| 精度 | 中等（靠手腕幅度） | 高（可微调瞄准） |

## 技术方案

### IMU → 准星映射
```
按下按钮 → 记录参考姿态 (ref_roll, ref_pitch)
运动中   → 准星位置 += (roll - ref_roll) × 灵敏度
           ref_roll 持续更新（避免漂移）
准星     → 约束在圆环内（内径~外径之间）
扇区     → 根据准星相对圆心的角度计算
```

### 姿态跟踪策略
- **增量速率控制**：陀螺仪积分跟踪姿态变化，不是绝对姿态
- **自动归零**：松开扳机后姿态清零
- **死区**：微小抖动不影响准星位置
- **灵敏度可配置**：config.yaml 中调整

### 架构精简
```
ESP32 ──BLE──→  BLEInputProvider
                      ↓
              AimEngine (光标准星)
              │  - 准星位置 x, y
              │  - 扇区计算
              │  - 死区/灵敏度
              ├──→ OverlayUI (绘制高亮)
              └──→ ActionExecutor (松开执行)
```

## 启动 V2 前需要确认的问题

1. **准星移动方式**：
   - A) 绝对角度映射（指哪打哪，松开后回中）
   - B) 速率控制（倾斜速度控制移动方向，归零即停）
   - C) 增量位移（每次倾斜角度变化累加到位移）

2. **灵敏度风格**：
   - 小幅度微调即可扫过全部 12 项
   - 还是需要大幅度摆动才能从头扫到尾

3. **是否需要 F12 作为调试备用输入**（建议保留，编译时可开关）
