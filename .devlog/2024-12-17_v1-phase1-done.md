# 2024-12-17: v1 Phase 1.0 — 项目骨架 + 所有核心模块实现

## 操作人
AI

## 涉及版本
v1

## 做了什么
- 完成技术选型：全栈 Python（核心） + tkinter（UI）+ YAML（配置）
- 搭建 v1 完整目录结构（8 个包，符合框架文档 Section 7.3）
- 实现所有核心模块：

  ### input/protocol.py (~120 行)
  - InputEvent、EventType、ButtonRole 定义
  - InputProvider 抽象基类（双向：read_events + feedback）
  - DeviceFeedback / FeedbackCommand 接口
  - NoopFeedback 空实现
  - 完全符合 DEVELOPMENT_FRAMEWORK.md Section 5.1~5.3

  ### input/mouse.py (~110 行)
  - MouseInputProvider 实现
  - put_button_down/up/motion/scroll 供 tkinter 回调调用
  - 鼠标位移 → 模拟姿态角（roll/pitch）
  - asyncio.Queue 事件队列（未来 BLE 接入也用同一机制）

  ### mapper/mapper.py (~100 行)
  - ActionMapper 类，config-driven 路由
  - ButtonRoute（overlay / direct）+ ScrollRoute
  - MenuItem dataclass

  ### gesture/engine.py (~80 行)
  - GestureEngine：死区滤波 → 灵敏度 → 加速度映射 → 菜单索引
  - GestureOutput 包含 cursor_index / delta / confidence

  ### ui/overlay.py (~220 行)
  - OverlayUI：tkinter 透明全屏窗口
  - Windows 穿透点击（WS_EX_TRANSPARENT）
  - 闲置时 alpha=0.01，激活时显示菜单
  - Canvas 绘制的深色主题菜单 + 高亮条
  - 10ms 轮询预留（未来 BLE 事件）

  ### executor/actions.py (~180 行)
  - ActionExecutor：跨平台键盘模拟
  - Win: PowerShell SendKeys
  - Mac: osascript
  - Linux: xdotool
  - 宏系统 + 脚本执行

  ### utils/config.py (~150 行)
  - AppConfig / MouseConfig / GestureConfig 等 dataclass
  - YAML 加载 + 校验

  ### main.py (~110 行)
  - AppController 串联所有模块
  - 配置加载 → 模块初始化 → 事件绑定 → UI 启动
  - 完整 5 个交互通道：左键菜单 / 右键截图 / F1 宏 / F2 静音 / 滚轮音量

- 安装 pyyaml 依赖
- 冒烟测试：全部模块导入通过 + 配置加载验证（4 按钮正确读取）

## 关键决策
- 放弃 Tauri（因 Rust 未安装），v1 全栈 Python + tkinter
- overlay.py 同时承担事件捕获和 UI 展示，InputProvider 接口保留但不直接挂载（Phase 2 再激活）
- 键盘模拟用平台原生工具而非 pyautogui（减少依赖）
