ESP32 + MPU6050 手枪式握把菜单选择，采用“激活时锁定基准”方案。

【核心逻辑】
- 用户按下按钮时，锁定当前姿态为“零点”
- 激活期间，根据手腕相对于零点的偏移角度，实时计算当前选中的扇区
- 松开按钮时，确认当前扇区并输出

【硬件配置】
- 传感器立姿握持（Z轴水平向前，X轴竖直向上）
- 一个按钮（连接GPIO，按下为LOW或HIGH可配置）
- BLE发送数据到Windows电脑

【算法要求】

1. 姿态解算：
   - 使用Madgwick滤波，输出四元数，频率100Hz

2. 激活锁定（按钮按下时）：
   - 记录当前四元数为 q_zero
   - 设置激活标志 is_activated = true

3. 激活期间（is_activated == true）的循环处理：
   a. 计算相对四元数：q_rel = q_zero^(-1) * q_current
   b. 从 q_rel 提取水平指向角（不受设备侧倾影响）：
      - 定义前向向量 [0,0,1] 旋转到世界坐标
      - 投影到水平面，计算 atan2(y, x) 得到角度 angle（-180 到 +180）
   c. 将 angle 映射到扇区：
      - 灵敏度参数 max_angle（默认60°），将 [-max_angle, +max_angle] 映射到 [-180, 180]
      - 根据扇区数量 N（可配置6/8/12）计算当前扇区索引
      - 实现迟滞逻辑（迟滞角度5°）防止边界抖动
   d. 通过BLE发送当前扇区索引

4. 确认（按钮松开时）：
   - 记录当前扇区为确认扇区
   - 发送确认包（包含最终扇区索引）
   - 清除激活标志 is_activated = false

5. BLE数据包格式：
   - 激活期间：0xAA + 扇区索引(1B)
   - 确认时：0xBB + 确认扇区(1B)
   - 发送频率：激活期间100Hz，待机时不发送或低频发送

【代码要求】
- 使用双核FreeRTOS：核心0传感器读取+算法，核心1 BLE发送
- 按钮使用中断或轮询（轮询间隔10ms）
- 删除所有delay和串口打印
- 输出完整Arduino代码
